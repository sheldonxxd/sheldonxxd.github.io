利用github-pages搭建Hexo博客

Created2020-12-20Updated2020-12-21

发现我的ubuntu主机无法连接到eduroam的无线网，而连接sjtu又会无法访问gitlab，所以这个事情目前我能想到的办法，就是让这台主机用无线网卡连接SJTU，用有限网卡连接一个无线路由器，然后大家如果要访问gitlab，就先连接路由器的WiFi，然后输入固定IP地址。为了验证这个思路，我打算现在自己家里尝试一下，使用hexo构建一个轻博客网站。

在家搭建局域网Hexo网站

先看Hexo的文档：

https://hexo.io/zh-cn/docs/

然后安装一下nodejs，才能使用npm命令：

https://github.com/nodesource/distributions#debinstall

尝试了一下，可能是因为在deepin里边，我直接用以下命令即可安装：

sudo apt-get install npm

结果这样安装的nodejs是10.19的版本，还好还好，勉强可以搞。

但是在调用npm的时候报错，所以还是尝试upgrade了一下nodejs，发现是可以的。

update是更新软件列表，upgrade是更新软件

但是我感觉自己执行命令之后，似乎把所有的软件都更新了，哎。然后版本并没有变化。所以我就还是安装着把！如果下一次要弄，我还是选择ubuntu18.04，因为u-net实际上只支持到这个版本，jupyterhub也是推荐这个版本。

然后运行一下命令安装hexo：

sudo npm npm install -g hexo-cli

然后把hexo添加到系统环境变量中：

echo 'PATH="$PATH:./node_modules/.bin"' >> ~/.profile

接下来新建一个目录，然后使用hexo命令进行初始化：

但是由于网速不好，这里半天都不动弹。本来我是想安装jupyterhub的，但是也是因为网络因素，无法翻墙，所以就没能成功。

接着就是进入目录内，进行安装。

cd hexo-website

npm install

接下来就是到_config.yaml文件中修改网站配置了。具体可以参考：

https://hexo.io/zh-cn/docs/configuration

但是对我来说，仅仅是想验证一下，所以把网址修改为自己的固定ip地址，然后从局域网其他设备访问这个地址试一下。

然后运行命令，开启网站服务器：

hexo s

然后就可以在手机浏览器中输入地址

http://192.168.0.20:4000/即可访问，真棒！

但是需要注意的是，这种策略要求client连接到局域网路由，如果路由器没有连接外网，在这段时间内就不能访问外网了。

将内容部署到github的page上

https://hexo.io/zh-cn/docs/one-command-deployment

建议使用git一键部署，然后要在本地已经存储git的用户名和密码，你只要git pull下来一个github的自己的仓库，然后git push一下即可。然后git config set一下user.name和user.email。如果这些基本信息没有保存，就会在depoly的时候报错，说https不支持。
